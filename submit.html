Verification Process

Credence uses ThinkTank's multi-agent security analysis to provide comprehensive evaluation of MCP servers. Our process combines automated security tools with AI-powered interpretation to catch vulnerabilities that traditional scanners miss.

üéØ Current Status: We're accepting submissions for the initial batch of MCP servers. Analysis turnaround time is currently 24-48 hours.

Submit Your Server

Provide your MCP server's GitHub repository URL and basic metadata. We'll automatically fetch the source code and dependencies.

2

Automated Security Analysis

Your server goes through our security pipeline:

  • Static code analysis (Semgrep, Bandit)
  • Dependency vulnerability scanning (Trivy, Syft)
  • Secret detection (GitLeaks)
  • SBOM generation
3

ThinkTank Multi-Agent Review

Our multi-agent swarm deliberates on the security findings:

  • 7 Believer agents: Emphasize server capabilities and safe patterns
  • 7 Skeptic agents: Challenge assumptions and identify risks
  • 5 Neutral agents: Synthesize perspectives and weigh evidence
  • 1 Foreperson: Generates final consensus report
4

Cryptographic Attestation

Verification results are signed using Sigstore and logged to Rekor's transparency log, creating an immutable record that can be independently verified.

5

Registry Publication & Badge

Verified servers receive:

  • Public listing in the Credence Registry
  • Embeddable verification badge for your README
  • Detailed security analysis report
  • Trust score and recommendations

What We Analyze

Security Checks

  • Code Quality: Potential bugs, security antipatterns, injection vulnerabilities
  • Dependencies: Known CVEs, outdated packages, malicious packages
  • Secrets: Hardcoded credentials, API keys, private keys
  • Configuration: Insecure defaults, excessive permissions
  • Behavioral Analysis: Suspicious network calls, filesystem access patterns

Supply Chain Verification

  • SBOM (Software Bill of Materials) generation
  • Dependency graph analysis
  • License compliance checking
  • Build reproducibility assessment
